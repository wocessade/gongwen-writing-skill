# 15 种公文文种模板

根据《党政机关公文处理工作条例》（中办发〔2012〕14号），公文种类主要有15种。

---

## 1. 决议

**定义:** 适用于会议讨论通过的重大决策事项。
**必备元素:** 发文机关标志、发文字号、标题、正文（含决议事项）、成文日期
**结构模板:**
```
标题: [发文机关]关于[事项]的决议
正文:
  一、决议背景与依据
  二、决议事项
  三、决议要求
发文机关: [署名]
日期: [成文日期]
```

---

## 2. 决定

**定义:** 适用于对重要事项作出决策和部署、奖惩有关单位和人员、变更或者撤销下级机关不适当的决定事项。
**必备元素:** 发文机关标志、发文字号、标题、正文（含决定事项）、发文机关署名、成文日期
**结构模板:**
```
标题: [发文机关]关于[事项]的决定
正文:
  一、决定缘由
  二、决定事项
  三、执行要求
发文机关: [署名]
日期: [成文日期]
```

---

## 3. 命令(令)

**定义:** 适用于公布行政法规和规章、宣布施行重大强制性措施、批准授予和晋升衔级、嘉奖有关单位及人员。
**必备元素:** 发文机关标志、发文字号（或令号）、标题、正文、签发人、成文日期
**结构模板:**
```
标题: [发文机关][文号]令
正文:
  公布[法规/规章名称]已经[会议]通过，现予公布，自[日期]起施行。
签发人: [姓名]
日期: [成文日期]
```

---

## 4. 公报

**定义:** 适用于公布重要决定或者重大事项。
**必备元素:** 发文机关标志、标题、正文、成文日期
**结构模板:**
```
标题: [发文机关][事项]公报
正文:
  一、会议/事项基本情况
  二、重要决定
  三、后续安排
日期: [成文日期]
```

---

## 5. 公告

**定义:** 适用于向国内外宣布重要事项或者法定事项。
**必备元素:** 发文机关标志、发文字号、标题、正文、成文日期
**结构模板:**
```
标题: [发文机关][事项]公告
正文:
  一、公告依据
  二、公告事项
  三、其他说明
日期: [成文日期]
```

---

## 6. 通告

**定义:** 适用于在一定范围内公布应当遵守或者周知的事项。
**必备元素:** 发文机关标志、发文字号、标题、正文、成文日期
**结构模板:**
```
标题: [发文机关]关于[事项]的通告
正文:
  一、通告背景与依据
  二、具体事项（可分段并列）
  三、实施时间与要求
日期: [成文日期]
```

---

## 7. 意见

**定义:** 适用于对重要问题提出见解和处理办法。
**必备元素:** 发文机关标志、发文字号、标题、主送机关、正文、发文机关署名、成文日期
**结构模板:**
```
标题: [发文机关]关于[事项]的意见
主送机关: [受理机关]
正文:
  一、总体要求
    （一）指导思想
    （二）基本原则
    （三）主要目标
  二、重点任务
    （一）[任务一]
    （二）[任务二]
    （三）[任务三]
  三、保障措施
发文机关: [署名]
日期: [成文日期]
```

---

## 8. 通知

**定义:** 适用于发布、传达要求下级机关执行和有关单位周知或者执行的事项，批转、转发公文。
**必备元素:** 发文机关标志、发文字号、标题、主送机关、正文、发文机关署名、成文日期
**结构模板:**
```
标题: [发文机关]关于[事项]的通知
主送机关: [各有关单位：]
正文:
  一、通知事由
  二、具体事项（可分段）
  三、工作要求
附件：[附件名称]（可选）
发文机关: [署名]
日期: [成文日期]
```

---

## 9. 通报

**定义:** 适用于表彰先进、批评错误、传达重要精神和告知重要情况。
**必备元素:** 发文机关标志、发文字号、标题、主送机关、正文、发文机关署名、成文日期
**结构模板:**
```
标题: [发文机关]关于[事项]的通报
主送机关: [各有关单位：]
正文:
  一、基本情况
  二、评价（或决定）
  三、希望与要求
附件：[附件名称]（可选）
发文机关: [署名]
日期: [成文日期]
```

---

## 10. 报告

**定义:** 适用于向上级机关汇报工作、反映情况，回复上级机关的询问。
**必备元素:** 发文机关标志、发文字号（含签发人）、标题、主送机关、正文、发文机关署名、成文日期
**结构模板:**
```
标题: [发文机关]关于[事项]的报告
主送机关: [上级机关名称：]
正文:
  一、工作开展情况
  二、主要成效（或问题）
  三、下一步工作打算
发文机关: [署名]
日期: [成文日期]
```

---

## 11. 请示

**定义:** 适用于向上级机关请求指示、批准。
**必备元素:** 发文机关标志、发文字号（含签发人）、标题、主送机关、正文（含请示事项）、发文机关署名、成文日期
**结构模板:**
```
标题: [发文机关]关于[事项]的请示
主送机关: [上级机关名称：]
正文:
  一、请示背景与缘由
  二、请示事项（具体明确）
  三、结语（"妥否，请批示"或"请予审批"）
发文机关: [署名]
日期: [成文日期]
```

---

## 12. 批复

**定义:** 适用于答复下级机关请示事项。
**必备元素:** 发文机关标志、发文字号、标题、主送机关（请示单位）、正文、发文机关署名、成文日期
**结构模板:**
```
标题: [发文机关]关于[事项]的批复
主送机关: [请示单位名称：]
正文:
  一、引述来文（"你单位《关于[事项]的请示》收悉"）
  二、批复意见（明确同意/不同意/部分同意及理由）
  三、工作要求
发文机关: [署名]
日期: [成文日期]
```

---

## 13. 议案

**定义:** 适用于各级人民政府按照法律程序向同级人民代表大会或者人民代表大会常务委员会提请审议事项。
**必备元素:** 发文机关标志、发文字号、标题、主送机关（人大或人大常委会）、正文、发文机关署名（政府首长）、成文日期
**结构模板:**
```
标题: [发文机关]关于提请审议[事项]的议案
主送机关: [人民代表大会/常务委员会：]
正文:
  一、议案背景与依据
  二、议案主要内容
  三、结语（"请予审议"）
发文机关署名: [政府首长姓名]
日期: [成文日期]
```

---

## 14. 函

**定义:** 适用于不相隶属机关之间商洽工作、询问和答复问题、请求批准和答复审批事项。
**必备元素:** 发文机关标志、发文字号、标题、主送机关、正文、发文机关署名、成文日期
**结构模板:**
```
标题: [发文机关]关于[事项]的函
主送机关: [主送机关名称：]
正文:
  一、发函缘由（或引述来函）
  二、商洽/询问/答复事项
  三、结语（"盼复"或"特此函复"）
发文机关: [署名]
日期: [成文日期]
```

---

## 15. 纪要

**定义:** 适用于记载会议主要情况和议定事项。
**必备元素:** 标题、正文、成文日期（可署名也可不署名）
**结构模板:**
```
标题: [会议名称]纪要
正文:
  一、会议基本情况（时间、地点、主持人、参会人员等）
  二、会议内容
    （一）[议题一]
    （二）[议题二]
  三、议定事项
  四、会议要求
发文机关/会议组织单位: [署名]（可选）
日期: [成文日期]
```

## 文种选用基本原则

| 行文方向 | 文种 | 特点 |
|---------|------|------|
| 上行文 | 报告、请示 | 下级→上级，需加签发人 |
| 下行文 | 通知、通报、批复、决定、决议、命令、意见、通告、公告、公报 | 上级→下级 |
| 平行文 | 函、意见 | 同级/不相隶属 |

**行文规则要点:**
- 不得在报告中夹带请示事项
- 请示应一文一事，不得多头主送
- 除上级机关负责人直接交办事项外，不得以本机关名义向上级机关负责人报送公文
- 同级机关必要时可以联合行文

## 事业单位高频公文场景指南

事业单位（公立学校、医院、科研院所、文化馆等）使用的公文文种与政府机关一致，但使用频率和侧重有所不同。

### 高频文种排名（按使用频率）

| 排名 | 文种 | 典型用途 |
|------|------|---------|
| 1 | 通知 | 人事任免、职称评审、年度考核、项目申报、会议通知 |
| 2 | 报告 | 年度工作报告、专项工作报告、整改报告 |
| 3 | 函 | 商洽合作、征求意见、申请审批 |
| 4 | 请示 | 请求批准编制、经费、项目 |
| 5 | 纪要 | 党委会/办公会/学术委员会会议纪要 |

### 任免通知 典型结构

```
一、任免依据（经XX会议/党委研究决定）
二、任职/免职事项（每人一行：姓名+职务）
三、到岗要求（任职时间、报到期限等）
```

### 职称评审通知 典型结构

```
一、评审范围（针对哪些专业技术岗位）
二、申报条件（基本条件+学历资历+业绩成果）
三、申报程序（个人申报→单位审核→主管部门审核）
四、材料清单（逐项列出）
五、时间安排（截止日期）
```

### 年度报告通知 典型结构

```
一、年报范围（所有登记的事业单位法人）
二、年报内容（年度报告书+财务报表+资质证明）
三、报送时间及方式（在线系统+纸质+截止日期）
四、工作要求（真实性+按时报送+责任追究）
```

### 商洽函 典型结构

```
一、发函缘由（简要说明背景）
二、商洽事项（逐条列出需要对方配合的事项）
三、时间要求（如有）
四、结语（'盼复'或'请予支持为盼'）
```

### 会议纪要 典型结构

```
一、会议基本情况（时间、地点、主持人、参会人员）
二、会议内容（分议题逐条记录讨论内容）
三、议定事项（逐条列出会议作出的决定）
四、工作要求（责任分工+时限要求）
```

### 自动化纪要注意事项

本技能支持 `--minutes-sample` 自动生成符合 GB 标准的纪要文档。自动生成时注意：
- **红头/文号自动跳过**：document_type="纪要"时，脚本自动跳过红头、发文字号、签发人
- **会议信息块自动插入**：通过 meeting_info dict（含 time/location/chair/attendees/recorder）自动渲染
- **主送机关可选**：纪要一般不分发主送，main_recipient 设为 null 即可跳过
- example: `python scripts/generate_gongwen_docx.py --minutes-sample`
