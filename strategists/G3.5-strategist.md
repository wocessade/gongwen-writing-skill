# Stage G3.5: 内容合规审查 [Strategist — 条件触发]

**Gate:** QG3.5 (ADVISORY — 可配置为 BLOCK)
**Goal:** 对 G3 格式审查通过的 .docx 进行内容事实核查，检测 AI 幻觉
**Needs Composers:** none

## Trigger Detection

读取用户消息 + session trigger_intent 变量：

1. **主触发词匹配**（精确匹配即无条件激活）：
   ```
   审稿 | 合规审查 | 内容核查 | 事实核查 | 合规检查 | 内容审查
   ```

2. **上下文触发词匹配**（含子串即激活，用于隐性要求）：
   ```
   审查内容 | 核实文中 | 检查政策引用 | 核对数据 | 有没有幻觉
   ```

3. **模式触发**：`mode=semi-auto` 时强制激活

## Agent Dispatch

**Mode B:** `content_compliance` agent × 3，中位聚合。

每个 agent 独立审查 6 维（policy_name, document_number, organization_name, personnel_title, statistics, legal_citation），输出结构化报告。

**聚合规则：**
- 2/3 一致 → 采用一致结果
- 1/3 异议 → 标记为 "contested"，不升级
- 3/3 全不一致 → 标记为 "contested"

## Gate QG3.5

```
if 聚合后无 Critical 发现:
    PASS → DO NOT route to G4
    输出内容合规审查报告（无问题项）
else:
    输出结构化问题列表 → route to G4 (仅 severity=Critical 和 Major 的项)
    附带 source_trail 供 G4 修复参考
```

### Gate Failure Route

```
if 用户没有明确要求审查，且触发词为上下文匹配:
    → ADVISORY — 提示用户："检测到公文含需要核实的内容，是否运行内容合规审查？"
    → 用户确认后运行 G3.5
    → 用户拒绝则 SKIP

if G3 审查未通过（未进入此阶段）:
    → 不触发 — 格式问题优先修复

if G3.5 审查发现 Critical 问题:
    → 输出问题列表到 G4
    → 标记 source_claim_map 到 passport
```
