# Stage G0: 前置 — 文种选择 + 上下文收集 [Strategist]

**Gate:** QG0 (SOFT_BLOCK)
**Goal:** 确认公文文种，收集发文机关/主送/密级/紧急程度等元数据
**Needs Composers:** none

## Decisions

### Axis 1: 文种识别

从用户输入中识别15种法定公文文种。若模糊则 STOP-AND-ASK，列出候选选项。

```
15种法定文种:
  - 上行文: 报告, 请示
  - 下行文: 通知, 批复, 通报, 决定, 决议, 命令, 意见, 通告, 公告, 公报
  - 平行文: 函
  - 兼类: 纪要

识别规则:
  if 用户明确提及文种名称 (e.g., "写一份通知"):
      → 直接确认 documentType = 该文种
  elif 用户描述场景但未提到文种 (e.g., "向上级申请经费"):
      → 根据场景推断文种
      → 推断置信度 >= 0.8: 确认并告知用户
      → 推断置信度 < 0.8:
          → STOP-AND-ASK
          → 列出 2-3 个候选文种及适用场景
          → 用户选择后确认
  elif 用户提供 .docx 文件:
      → 从文件名/正文推断文种 (如有红头则读取)
      → 告知用户识别结果并确认
```

### Axis 2: 上下文完整性

收集并验证必填字段。缺失时主动询问。

```
必填字段:
  - 发文机关: 必须确认 (e.g., "XX省人民政府办公厅")
  - 标题主题: 必须确认 (e.g., "关于做好2024年元旦春节期间有关工作")
  - 主送机关: 必须确认 (e.g., "各市、县人民政府，省政府各部门")

条件字段:
  if documentType in [报告, 请示]:  # 上行文
      - 签发人: 必须询问并确认
      - 附注(联系人): 推荐询问

补充字段 (可选):
  - 抄送机关
  - 印发日期
  - 印发机关
  - 共印份数
```

### Axis 3: 密级与紧急程度

仅当用户提及或文种适用时标注。

```
密级:
  values: [none, 秘密, 机密, 绝密]
  default: none
  触发: 用户提到"密级"/"涉密"/"机密"
  规则: 上行文可用密级；下行文如涉及国家安全才标注

紧急程度:
  values: [normal, 特急, 加急]
  default: normal
  触发: 用户提到"紧急"/"加急"/"特急"
  规则: 下行文可用紧急程度；上行文不使用
```

### Axis 4: 模式选择

```
if 用户要求逐阶段确认:
    → mode = semi-auto
    → 每阶段门控暂停，等待用户确认
elif 用户要求全自动:
    → mode = auto
    → 仅门控失败时询问
else:
    → mode = auto (default)
    → 门控失败时 STOP-AND-ASK
```

### Axis 5: 输入方向判断

```
if 用户提供已有 .docx 文件要求审查:
    → entry_point = existing-manuscript
    → 直接跳转至 G3 (格式审查)
else:
    → entry_point = new-document
    → 继续 G1 (结构规划)
```

## Gate

### QG0 Gate Checklist

- [ ] 文种已确认（15种法定文种之一）
- [ ] 发文机关已确认
- [ ] 标题主题已确认
- [ ] 主送机关已确认
- [ ] 上行文 (报告/请示): 签发人已确认
- [ ] 密级与紧急程度已核实（如适用）
- [ ] 模式 (auto/semi-auto) 已确定
- [ ] 输入方向 (new-document/existing-manuscript) 已确定

### Gate Failure Route

```
if 文种未确认:
    → SOFT BLOCK — 列出候选文种要求用户选择
    → 用户选择后继续
if 发文机关/标题主题/主送机关缺失:
    → SOFT BLOCK — 逐项询问用户
    → 全部填写后继续
if 上行文且签发人缺失:
    → SOFT BLOCK — 询问签发人姓名
    → 填写后继续
if 所有必填字段已填写:
    → PASS — 传递 context 至下一阶段
```

**To Next Stage:** 文种类型、发文机关、标题主题、主送机关、签发人(如有)、密级、紧急程度、模式、输出路径 → G1
