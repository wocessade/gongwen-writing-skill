# Stage G2: 起草 .docx [Strategist]

**Gate:** QG2 (BLOCK)
**Goal:** 运行 generate_gongwen_docx.py 生成符合 GB/T 9704 的 .docx
**Needs Composers:** [gongwen-docx-assembly]

## Critical Rule

**必须通过脚本生成，不得在对话上下文中直接撰写公文内容。**

- 禁止在对话中手动写出完整的公文正文
- 必须以 Python 脚本 (python-docx) 操作 XML 层生成 .docx
- 脚本可调用已有函数或完全重写

## Decisions

### Axis 1: 生成模式选择

```
if G0 阶段已收集完整正文内容:
    → 直接生成: 所有内容在脚本中硬编码或从结构化数据传入
elif 正文内容需要组合:
    → 组合生成: 从 G1 核心信息点收集用户提供的内容，结构化组织后写入脚本
elif 用户提供参考文档 (.docx 示例):
    → 参考生成: 加载参考文档分析格式 → 按相同格式生成
```

### Axis 2: 脚本结构规划

```
脚本必须包含以下功能模块:
  1. 页面设置模块:
     - 纸张: A4 (210mm × 297mm)
     - 版心: 左 28mm ± 1mm, 右 26mm ± 1mm, 上 37mm ± 1mm, 下 35mm ± 1mm
     - 装订线: 0 (公文不设装订线)
  2. 红头模块 (下行文 / non-纪要):
     - 发行机关标志: 方正小标宋简体 / 小标宋, 红色, 居中
     - 分隔线: 红色, 与版心等宽
     - 注: 纪要文种不设红头和发文字号
  3. 发文字号模块:
     - 格式: {机关代字}[{年份}]{顺序号}
     - 字体: 仿宋_GB2312 / 仿宋, 居中
     - 注: 纪要文种跳过发文字号
  4. 标题模块:
     - 字体: 方正小标宋简体 / 小标宋, 居中
     - 行距: 2.0 倍行距
  5. 主送机关模块:
     - 字体: 仿宋_GB2312 / 仿宋, 顶格
  6. 正文模块:
     - 字体: 仿宋_GB2312 / 仿宋, 三号 (16pt)
     - 行距: 1.5 倍 (固定值 28pt)
     - 首行缩进: 2 字符
     - 一级标题: 黑体, 三号 (16pt)
     - 二级标题: 楷体_GB2312 / 楷体, 三号 (16pt)
  7. 附件标记模块 (如有):
     - 正文末尾空一行后标注
  8. 发文机关落款 + 成文日期模块
```

### Axis 3: 字体回退策略

```
字体加载优先级:
  primary:
      - 发行机关标志: 方正小标宋简体 → 方正小标宋 → 宋体
      - 正文: 仿宋_GB2312 → 仿宋 → 宋体
      - 一级标题: 黑体
      - 二级标题: 楷体_GB2312 → 楷体 → 宋体
  fallback:
      if 首选字体未安装:
          → 使用回退字体
          → 在脚本开头输出警告: "[WARNING] 系统未安装{字体名}，已回退至{回退字体}"
```

## Protocol

1. **加载 composer:** 加载 `composers/gongwen-docx-assembly.md` 获取模板和组装规范
2. **加载 GB/T 9704 参考:** 加载 `references/gb-t9704-2012-quick-ref.md` 查表校验尺寸/字体/行距
3. **构建 config dict:**
   ```python
   config = {
       "documentType": "通知",       # 从 G0
       "organ": "XX省人民政府办公厅",  # 从 G0
       "title": "关于...的通知",       # 从 G0
       "mainSend": "各市...",        # 从 G0
       "signer": "张三",             # 上行文时从 G0
       "body_sections": [...],      # 从 G1 结构化信息点
       "attachments": [...],        # 从 G1 (可选)
       "securityLevel": "none",     # 从 G0
       "urgency": "normal",         # 从 G0
       "output_path": "..."         # 计算输出路径
   }
   ```
4. **编写并执行脚本:**
   - 以 `generate_gongwen_docx.py` 为文件名写入 `scripts/` 或 `{output_dir}/`
   - 可调用 `scripts/` 中已有的辅助函数，或在脚本中从零编写
   - 使用 `subprocess.run([sys.executable, "脚本路径"])` 执行
   - 捕获 stdout/stderr 确认执行成功
5. **确认生成:**
   - 验证 .docx 文件存在且文件大小大于 0
   - 读取 .docx 文件头确认不是空文档
   - 输出生成路径供用户验证

## Gate

### QG2 Gate Checklist

- [ ] .docx 已成功生成，文件路径可访问
- [ ] 未跳过脚本直接撰写公文内容
- [ ] 红头 + 分隔线 (下行文/non-纪要) 已渲染
- [ ] 字体、字号、行距符合 GB/T 9704 标准
- [ ] 页面设置 (版心尺寸) 正确
- [ ] 发文字号格式正确 (纪要跳过)
- [ ] 附件 (如有) 已标注
- [ ] 脚本执行无未捕获异常

### Gate Failure Route

```
if .docx 未生成或文件为空:
    → BLOCK — 检查脚本错误
    → 修复后重新执行
if 发现直接在对话中撰写了公文内容:
    → BLOCK — 必须使用脚本生成
    → 删除已撰写的对话内容，转用脚本
if 格式严重偏离 GB/T 9704 (e.g., 字体错误、页边距错误):
    → BLOCK — 修正脚本后重新生成
if 脚本执行报错:
    → BLOCK — 分析错误信息并修复
    → 最多重试 3 次
    → 3 次失败后 STOP-AND-ASK
```
